This page is the stable product canon for autonomous project work in this repository.

Workers should use this page to understand what the project is trying to achieve before they interpret
an active PR page.

## Users

- founder-operators
- very small AI-native teams
- builders who want one durable company runtime instead of a pile of disconnected agent sessions

## Core job to be done

Help a human operator run a durable business through autonomous agents without losing accountability,
continuity, or governance.

## Direction

openboa is building a Business as Agent runtime.
The business is the durable operating subject.
Agents are an evolvable workforce coordinated through shared truth.

The current shipping wedge remains chat-first.

## Value promise

- durable business identity
- shared company memory
- accountable autonomous execution
- one believable coordination surface for humans and agents

## Current wedge

- credible multi-agent company chat
- channels, DMs, group DMs, threads, mentions, unread, and search
- human and agent participation in one shared room model

## Out of scope for the current wedge

- a first-class Work board as the primary shipped surface
- a first-class Observe product surface as the primary shipped surface
- dashboard-first control-plane UX
- feature volume as a substitute for believable core behavior

## Product-fit bar for current PR work

A PR is product-aligned only if it makes the current wedge more believable, more durable, or more
governable without breaking the chat-first thesis.
