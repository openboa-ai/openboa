<AccordionGroup>
  <Accordion title="What is openboa?">
    openboa is an open-source runtime for Business as Agent (BOA), where a human operator governs durable business execution performed by autonomous agents.
  </Accordion>
  <Accordion title="What can openboa do today?">
    openboa is still early and design-first. The current value is a shared conceptual model, governance framing, and documentation foundation.
  </Accordion>
  <Accordion title="What is not available yet?">
    A production-ready runtime stack, mature orchestration layer, and full CLI workflow surface are not published yet.
  </Accordion>
  <Accordion title="Who is this for?">
    Builders who want to run agent-based execution while retaining policy control, memory continuity, and auditability.
  </Accordion>
  <Accordion title="Why docs first?">
    Docs-first development keeps primitives, constraints, and vocabulary aligned before implementation complexity grows.
  </Accordion>
  <Accordion title="Is this a generic chatbot framework?">
    No. openboa focuses on the business as the durable product unit, not a single conversational bot.
  </Accordion>
  <Accordion title="Is a human always in control?">
    Yes. Human governance is a core requirement, especially for approvals, policy boundaries, and safety decisions.
  </Accordion>
  <Accordion title="How should I start reading?">
    Start with BOA concepts, then color/system foundations, then operational pages and troubleshooting.
  </Accordion>
</AccordionGroup>
