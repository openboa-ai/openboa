# Brain Openboa

## Role

`brain-openboa` is an optional repo-specific framing utility for openboa product meaning, wedge, primitives, governance, roadmap semantics, and IA.

## When to use

Use it when the task is about what openboa means, how maintained openboa docs should be consolidated, or where repo-local product meaning should live next.

## What it optimizes for

- sharper repo-local framing
- consolidation of overlapping meaning
- clean routing into `wiki/syntheses/`, `wiki/prs/`, or `docs/`

## Required outputs

- the core thesis or clarified meaning
- key tensions and tradeoffs
- repo-local implications
- the recommended next artifact or owner

## Stop condition

Stop when the framing question is materially clearer and the next owning artifact is explicit.

## Handoff expectation

Hand back to `auto-project`, `auto-pm`, or `auto-wiki` when the framing result must become a frontier, a memory update, or stable canon.

## Hard boundaries

- optional utility only; not part of the default harness loop
- do not replace `auto-wiki`, `auto-pm`, `auto-coding`, `auto-qa`, or `auto-ui`
- read maintained repo docs before raw sources
- do not treat external philosophy as repo-local truth
- do not jump from framing directly to code if a PR frontier is not locked
